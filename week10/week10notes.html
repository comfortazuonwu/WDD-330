Validating API

  • Client-side validation is the first step to check for invalid data.
  • Security checks should be performed on the server side and the client side as well
  • Checking if the data entered in the form is the correct format and within the constraints set by the application is known as form validation.
  • Constraint validation API enables checking values that users have entered into form controls before submitting the value to the server.

FETCH API

  • The Fetch API provides a Javascript interface for accessing and manipulating parts of the HTTP pipeline
  • - The fetch() method provides an easy way to fetch resources asynchronously across the network
  • - The fetch() method can also be used to upload JSON data, upload multiple files and process a text file line by line
